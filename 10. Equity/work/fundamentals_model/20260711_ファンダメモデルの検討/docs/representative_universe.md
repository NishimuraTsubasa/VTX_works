# 代表銘柄ユニバース選定ロジック

## 1. 目的

ファクターユニバースが実際の指数構成銘柄を完全には網羅しない場合に、指定銘柄数で指数のセクター構成とリターン特性を近似する代表銘柄集合を作る。

## 2. 判定順序

1. 指数と国・通貨キーを紐付ける
2. 当該キーに属する利用可能銘柄を抽出する
3. ファクターカバレッジ不足銘柄を除外する
4. 目標セクターウェイトからセクター別銘柄数を配分する
5. 各セクター内で代表性と重複度を評価して銘柄を選ぶ
6. セクターウェイト制約下でトラッキングウェイトを推定する
7. 選定数、セクターカバレッジ、追随相関、RMSEを保存する

## 3. 選定銘柄数

`target_count_by_index` を優先し、未指定の場合は `target_count_by_key` を使用する。

## 4. セクター配分

最大剰余法により整数配分し、利用可能銘柄数を上限として不足分を他セクターへ再配分する。

## 5. 銘柄選定指標

- 過去指数リターンとの正相関
- リターン履歴カバレッジ
- 時価総額順位
- 実指数構成銘柄フラグ
- 選定済み銘柄との最大相関

## 6. ウェイト推定

非負・セクター合計制約付き最小二乗問題をSLSQPで解く。Ridgeペナルティにより少数銘柄への過度な集中を抑える。

## 7. 品質指標

- `selected_count / target_count`
- `sector_weight_coverage`
- `tracking_correlation`
- `tracking_rmse`
- `actual_constituent_share`
- `optimizer_success`
